Rootin Tootin Robot

A fun gun game
A Project By Derek Lee, Keyi Zhou.

Demonstration Video

Introduction

We present a shooting robot game that we designed and built using Raspberry Pi, PiTFT, motors, servos, and Bluetooth communication. Our objective was to create an engaging and fun game and demostrate what we have learnt throughout the quarter, and that can be played by one or two players. Here will describe the design, testing, and results of our project.

Project Objective:

  • Design and build a shooting robot game using RaspberryPi
  • Wire and assemble a robot containing motors, a Raspberry Pi, a PiTFT, wheels, servos and power supplies.
  • 3D design and print a gun model that can contain a Raspberry Pi, a recovery button, a power bank, and a camera.
  • Control the servo and motor using PWM signal
  • Implement computer vision algorithms to detect colors in the camera's field of view.
  • Program the gun device to communicate wirelessly with the robot car using Bluetooth.

Design

We used Raspberry Pi as the main controller for both the robot car and the gun station. The robot car uses two motors, a motor controller and controlled by hardware PWM, and had two balloon flags controlled by servos controlled by software PWM. The gun station had a camera and use color detection to detect whether the target is aimed and a recovery button for triggering the detection process and starting/stoping the game. The two components communicated with each other via Bluetooth.

When the button from gun is being pressed for the first time, the balloon flags will raise indicating the start of the game. Then the robot will move randomly. Player should try to aim the camera at the balloon flag and use the button to "shoot". When the button is pressed, the camera will take a picture, then analyze the center color of the image. If the color turns out to be one of the flag's, it will send a message to the robot. Robot will put down the flag by rotating the servo. When both targets are down, player won the game, and the robot will stop. Pressing the button once agian will start the game agian. After the first target being down, player has 5 chances to shoot down the other one. Failing to do so will result in the restart of the game, which both flag will be raised again. There is also an emergency stop flag, which is green, when button is pressed and camera detected green, the robot will stop. Pressing the button again will restart the game.

Testing

We tested our shooting robot game extensively to ensure that it worked as intended and was free of bugs and glitches. We performed multiple rounds of testing, including hardware and software tests for each seperate unit, and the overall playing experience. It truns out that the robot is working as expected. We also collected feedback from several test users to identify areas for improvement and address any issues that arose.

Result

Our shooting robot game was successful in achieving its objectives of providing an engaging and fun game. We received positive feedback from test users and were able to identify areas for improvement to make the game even better.
